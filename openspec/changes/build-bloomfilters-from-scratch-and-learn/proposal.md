# Proposal: Build Bloomfilters from Scratch and Learn

## What & Why

This proposal to implement a Bloom filter from scratch to understand its inner workings and applications. Bloom filters are probabilistic data structures used for efficient membership testing, with applications in databases, caching, and distributed systems. Building one from scratch will deepen understanding of its design and trade-offs.

## Goals

- Implement a Bloom filter from scratch.
- Learn about hash functions and their role in probabilistic data structures.
- Explore use cases and limitations of Bloom filters.

## Non-Goals

- Optimizing for production-level performance.
- Implementing advanced variations like counting Bloom filters.

## Success Criteria

- A working implementation of a basic Bloom filter.
- Documentation explaining the design and implementation process.
- Examples demonstrating its use cases.

## Risks

- Misunderstanding the mathematical foundations.
- Overcomplicating the implementation.

## Next Steps

1. Design the Bloom filter.
2. Implement the core functionality.
3. Test with example use cases.
4. Document the process and findings.
